# 对标账号风格库

## 用途

记录对标账号的写作模式、标题套路、内容策略，供写作时参考借鉴。
不直接复制内容，而是提取**结构模式**和**表达手法**。

---

## 账号一：爱AI的大刘（Ai大刘）

**公众号**：爱AI的大刘
**定位**：AI Agent 工具的实操测评 + 保姆级教程
**目标人群**：技术爱好者、开发者、想用 AI 提效的职场人
**更新频率**：1-2 篇/周
**分析日期**：2026-05-05
**样本量**：6 篇（2026年3月-4月）

### 标题套路

| 类型 | 公式 | 示例 |
|------|------|------|
| 数字冲击 | 大数字 + 精选承诺 | "13000 个 skills，用这些就够了" |
| 情绪感叹 | 感叹词 + 工具名 + 核心卖点 | "绝了！Agent 自动选最适合的 Skill" |
| 等式公式 | 方案A + 方案B = 结果 | "Cowork + Deepseek V4 = 无敌" |
| 保姆级 | "保姆级教程"/"手把手" + 时间承诺 | "30分钟手把手教程" |
| 万字承诺 | "万字！" + 工具名 + 深度承诺 | "万字！Hermes Agent 最细部署指南" |

**核心公式**：情绪词 + 具体数字 + 工具名 + 承诺（教程/总结/指南）

**可借鉴点**：
- 标题带具体数字比泛泛描述更有说服力
- "保姆级""手把手"降低读者心理门槛
- 工具名出现在标题里，利于搜一搜搜索
- "万字"承诺暗示深度，吸引认真学习型读者

### 开头钩子模式

1. **数据冲击型**
   > 13,338 个 Skills。你一天试一个，不吃不喝不睡觉，你需要试 37 年。

   大数字 → 荒诞换算 → 具象比喻。适合工具盘点类。

2. **产品发布型**
   > 就在上周，Anthropic 公测 Claude Managed Agents——一个 AI 替你打工一小时，只要 8 美分。

   一句话概括新品 → 核心卖点定价 → 信息密度极高。

3. **共识质疑型**
   > 所有人都在说选对 Skills 是重点。但是不知道大家感觉到没，问题就出在选对这个事儿上。

   共识前置 → 转折质疑 → 痛点升级。

4. **信息差型**
   > 这两天 Anthropic 在桌面端悄悄塞了一个开关。

   "悄悄塞了"制造信息差 → 一句话解释技术含义 → 极简收束。

### 正文结构模板

```
H1 标题（情绪词+数字+工具名+承诺）

一句话概括（头图前的导语）
头图
✍️ 作者 / 📝 编辑 / 🎨 排版（签名栏）

正文第一段：痛点/数据/个人实测结论

## 一、小标题

01

一句话点题

正文 + 加粗结论 + 截图

## 二、小标题

02

一句话点题

正文...

（重复 3-6 个 H2）

## 写在最后 / 总结

CTA（关注系列/文末链接/评论区互动）
```

### 写作风格指纹

| 特征 | 做法 |
|------|------|
| 人称 | 第一人称为主，"我实测了一周"、"我自己深度使用" |
| 语气 | 热情但不油腻，像兴奋的朋友在分享发现 |
| 口语化 | "话不多说"、"泪目！"、"什么玩意都能上架" |
| 情绪表达 | 直接。"扎心的数据"、"最离谱的是" |
| 数据使用 | 大量具体数字，不说"很多"，说"60多个" |
| 权威感来源 | 个人实测 + 截图佐证 + 踩坑经历 |
| 配图风格 | 产品截图 + 标注箭头 + 数据图表，不搞氛围图 |

### 内容策略

**选题方向**：全围绕 AI Agent 生态
- 工具测评（新工具深度体验）
- 保姆级教程（部署/接入/配置）
- 精选合集（N 个最值得装的 XX）
- 安全警示（恶意工具/踩坑经历）

**系列化**：
- "Claude Code 一周精通系列" → 5 篇
- "OpenClaw 从零到神"系列
- 每篇文末互相导流

**差异化壁垒**：
- 亲身实测（"我试了 60 个，留了 30 个"）
- 踩坑叙事（API Key 被盗等真实经历）
- 安全视角（推荐工具的同时提醒风险）

### 可借鉴清单

| 优先级 | 技巧 | 说明 |
|--------|------|------|
| ⭐⭐⭐ | 标题带具体数字 | "13000 个"比"很多"有力 10 倍 |
| ⭐⭐⭐ | 每节加编号 + 一句话点题 | 给读者进度感，方便跳读 |
| ⭐⭐⭐ | 踩坑叙事 | "我替你踩了"比"注意事项"有说服力 |
| ⭐⭐ | 加粗关键结论 | 跳读读者也能抓重点 |
| ⭐⭐ | 签名栏 | 建立作者品牌，显得专业 |
| ⭐⭐ | 系列化运营 | 培养读者粘性，互相导流 |
| ⭐⭐ | 安全叙事 | 建立"靠谱"形象 |
| ⭐ | 引用块包装公式/定义 | 视觉上从正文中跳出来 |
| ⭐ | 截图密度 | 平均 8-14 张/篇，但注意不要照搬 |

## 账号二：木易的AI频道（AI信息Gap）

**公众号**：AI信息Gap
**作者背景**：Top2 + 美国 Top10 CS 硕，AI 产品经理
**定位**：AI 行业深度分析 + 产品解读 + 热点速递
**目标人群**：产品经理、技术决策者、想理解 AI 行业的职场人
**分析日期**：2026-05-05
**样本量**：3 篇（2026年5月）

### 标题套路

| 类型 | 公式 | 示例 |
|------|------|------|
| 热点速递 | 关键词冲上热搜 + 副标题 | "「豆包付费」冲上热搜！最高 500 元/月" |
| 反直觉警告 | 旧方法失效 + 权威来源 | "你以前精心写的提示词，可能对 GPT-5.5 不管用了" |
| 数字总结 | 权威发布 + 我总结了 N 个变化 | "OpenAI 官方发布 GPT 5.5 提示词指南，我总结了 7 个关键变化！" |
| 荒诞叙事 | 产品被XX附体 + 评价 | "GPT-5.5 被「哥布林」附体了！OpenAI 最荒诞的对齐翻车" |

**核心公式**：热点事件/权威来源 + 反直觉钩子 + 数字化承诺

### 开头钩子模式

1. **热搜体**
   > 「豆包付费」。这几个字直接冲上了微博热搜。

   极短句 → 事实 → 冲击力。适合热点新闻。

2. **权威背书 + 反转**
   > 你以前精心写的提示词，可能对 GPT-5.5 不管用了。这是 OpenAI 自己说的。

   颠覆认知 → 立刻给权威来源 → 让读者不得不信。

3. **荒诞叙事**
   > OpenAI 的 GPT-5.5，被一群哥布林「附体」了。

   拟人化 + 荒诞感 → 好奇心拉满。

### 正文结构模板

```
H1 标题

导语（1-2 句话概括核心信息）
配图

正文展开（背景 → 核心信息 → 深度分析 → 行业对比 → 结论）

> 作者简介签名块
关注引导图
```

**特点**：
- **不加编号节**（和大刘不同），用自然段落推进
- **信息密度极高**，每段都有干货，不灌水
- **善用引用块**包装关键定义、公式、官方原话
- **数据对冲**：总是把 A 和 B 放在一起比，让读者自己判断
- **结尾克制**：不煽情，一句总结收束

### 写作风格指纹

| 特征 | 做法 |
|------|------|
| 人称 | 第一人称但克制，"我总结了"、"据知情人士透露" |
| 语气 | 理性、克制、有判断力但不强加。像一个靠谱的分析师朋友 |
| 口语化 | 适度。"也就是说"、"换句话说"、"严格来说" |
| 情绪表达 | 极克制。偶尔一句"恐怕是 AI 领域有史以来最荒诞的" |
| 数据使用 | 精确到小数点。"3.45 亿"、"1600 亿人民币"、"2.5 折" |
| 权威感来源 | 行业数据 + 官方来源 + 知名人士引用（Simon Willison、Sam Altman） |
| 对比手法 | 横向对比（ChatGPT vs Claude vs 豆包），纵向对比（历史 → 现在 → 未来） |

### 内容策略

**选题方向**：
- 热点事件速评（产品上线、定价变化、行业新闻）
- 官方文档解读（提示词指南、技术博客、API 变更）
- 行业格局分析（市场份额、竞争态势、商业模式）

**差异化壁垒**：
- 信息源优势（官方文档 + 行业人脉）
- 分析深度（不停留在新闻层面，挖到技术原理和商业逻辑）
- 克制风格（不煽情、不标题党，靠内容质量留人）

### 可借鉴清单

| 优先级 | 技巧 | 说明 |
|--------|------|------|
| ⭐⭐⭐ | 权威来源 + 反直觉开头 | "OpenAI 自己说的"比"我觉得"有力 |
| ⭐⭐⭐ | 数据精确到小数点 | "3.45 亿"比"3 亿多"可信 |
| ⭐⭐⭐ | 横向对比表 | 一张表胜过一千字描述 |
| ⭐⭐ | 引用块包装关键信息 | 官方原话、公式、定义用引用块 |
| ⭐⭐ | 签名块+引导关注 | 固定格式，品牌感强 |
| ⭐⭐ | 结尾克制收束 | 不煽情，一句有力的判断句 |
| ⭐ | 热搜体开头 | 极短句 → 事实 → 冲击力 |

## 账号三：越哥聊AI

**公众号**：越哥聊AI
**定位**：AI 工具教程 + 行业科普 + 产品接入指南
**目标人群**：AI 入门用户、想用 AI 的普通职场人、创业者
**分析日期**：2026-05-05
**样本量**：2 篇（2025年5月-2026年3月）

### 标题套路

| 类型 | 公式 | 示例 |
|------|------|------|
| 重磅产品 | 产品名 + 上线 + 功能承诺 | "微信官方小龙虾插件（ClawBot）上线！详细接入教程+ 全方案对比" |
| 万字深度 | "万字长文！" + 主题 + 全面承诺 | "万字长文！AI智能体全面爆发前夜：一文讲透技术架构与行业机会" |

**核心公式**：重磅感 + 详细教程/全面分析承诺

### 开头钩子模式

1. **直接宣布型**
   > 刚刚，微信正式放出了ClawBot（小龙虾）插件入口。

   "刚刚"制造紧迫感 → 一句话宣布事实 → 立刻给价值承诺。

2. **认知刷新型**
   > 你可能听说过：2023 是智能体元年，但很多人没意识到，2025年，才是智能体真正落地改变世界的第一年。

   旧认知 → 转折 → 新判断 → 建议收藏。

### 正文结构模板

```
H1 标题

导语（价值承诺 + 内容预告）
配图

## 一、教程/基础部分
## 二、核心内容
## 三、对比/分析
## 四、总结/展望

文末福利（回复关键词领取资料）
相关文章推荐列表
```

**特点**：
- **教程导向**：步骤清晰，截图丰富，零基础友好
- **福利驱动**：文末总有"回复 XX 领取资料"
- **推荐列表**：文末放 5-7 篇相关文章，互相导流
- **科普风格**：解释概念时喜欢用定义框 + 架构图

### 写作风格指纹

| 特征 | 做法 |
|------|------|
| 人称 | 第一人称，但偏教学口吻 |
| 语气 | 热情 + 耐心，像一个愿意教你的师兄 |
| 口语化 | 适度。"通俗理解"、"划重点"、"核心结论" |
| 数据使用 | 多用图表和对比表，少用文字描述数据 |
| 配图风格 | 大量架构图、对比表、流程图、截图，图文比约 1:1 |
| 收尾 | 总有 CTA：福利领取 + 关注 + 推荐阅读 |

### 内容策略

**选题方向**：
- 产品接入教程（微信插件、Coze、各种平台）
- 行业科普长文（智能体架构、技术发展史）
- 工具对比评测

**差异化壁垒**：
- 零基础友好（真的手把手，不是假装的）
- 福利驱动（资料包吸引关注和转发）
- 图文并茂（架构图让复杂概念可视化）

### 可借鉴清单

| 优先级 | 技巧 | 说明 |
|--------|------|------|
| ⭐⭐⭐ | 步骤教程截图密度 | 教程类每步都截图，零基础友好 |
| ⭐⭐⭐ | 文末福利钩子 | "回复 XX 领取"是被验证过的涨粉公式 |
| ⭐⭐ | 架构图/对比表 | 复杂概念用图比用文字高效 10 倍 |
| ⭐⭐ | 文末推荐阅读 | 5-7 篇相关文章互相导流 |
| ⭐⭐ | "刚刚"开头 | 制造紧迫感和时效性 |
| ⭐ | 定义框 | 概念解释用引用块或特殊格式 |

## 写作时如何用这份库

1. **选题阶段**：看三个账号的标题套路，找灵感
2. **框架阶段**：根据你的内容类型选对应的结构模板
3. **写作阶段**：
   - 想写热情分享型 → 参考大刘的风格指纹
   - 想写深度分析型 → 参考木易的风格指纹
   - 想写教程指南型 → 参考越哥的风格指纹
4. **收尾阶段**：根据你的目标选收尾方式（涨粉用福利，品牌用签名，深度用克制收束）
